配置与管理DNS服务器

liaoyl 发布于 2023-05-01

项目9 配置与管理DNS服务器

部署环境：

9.2.2 安装DNS服务器角色

在win2008-1下安装DNS服务器角色，步骤如下：

1. 以域管理员身份登录win2008-1，选择“开始”—“管理工具”—“服务器管理器”—“角色”选项，在控制台右侧单击“添加角色”，单击下一步，勾选“DNS服务器”复选框

2. 单击下一步，显示“DNS”服务器窗口，其中简述了DNS功能和注意事项

3. 单击下一步，出现“确认安装选择”窗口，在域控制器上安装DNS服务器角色，区域将与AD域服务集成在一起

4. 单击安装，完成DNS服务器角色安装

9.2.3 DNS服务器的停止和启动

1.使用net命令

以域管理员身份登录win2008-1，进入cmd命令行，输入net stop dns停止DNS服务;net start dns开启DNS服务

2. 使用“DNS管理器”控制台

单击“开始”—“管理工具”—“DNS”，打开DNS管理器控制台，在左侧右键单击服务器win2008-1，选择“所有任务”—“停止或启动或重新启动”，即可对DNS服务器进行控制

3. 使用“服务”控制台

单击“开始”—“管理工具”—“服务”，打开“服务”控制台，找到“DNS Server”服务，选择“启动”或“停止”操作即可启动或停止DNS服务器

9.3 部署主DNS服务器的DNS区域

在域控制器上安装完DNS服务器角色后，将存在AD域服务集成的区域，为实现以下实例，要将其删除

9.3.1 创建正向主要区域

在DNS服务器上创建正向主要区域“lyl26.com”，步骤如下

1. 在win2008-1上，单击“开始”—“管理工具”—“DNS”，打开DNS管理器控制台，展开DNS服务器目录树，右键单击“正向查找区域”选项，选择“新建区域”

  1. 单击下一步，出现“区域类型”窗口，用来选择要创建的区域类型，若要创建新的区域，要选中“主要区域”

3. 单击下一步，选择在网络上如何复制DNS数据，本例选择如图选项

4. 单击下一步，在“区域名称”中设置要创建区域名称，如lyl26.com，区域名称用于指定DNS名称空间的部分，由此实现DNS服务器管理

5. 单击下一步，选择“只允许安全的动态更新”选项

6. 单击下一步，显示新建区域摘要，单击“完成”按钮，完成区域创建

9.3.2 创建反向主要区域

反向查找区域用于通过IP地址来查询DNS名称，步骤如下：

1. 与上文创建正向查找区域相似，右键反向查找区域，选择“新建区域”，并在类型选择“主要区域”

2. 在“反向查找区域名称”窗口中，选择IPV4反向查找区域“选项，如图所示

  1. 单击下一步，在对话框中输入网络ID或反向查找区域名称，本例输入的是网络ID，区域名称根据网络ID自动生成，如图所示

4. 单击下一步，选择“只允许安全的动态更新”选项

5. 单击下一步，显示新建区域摘要，单击完成

9.3.3 创建资源记录

DNS服务器需要根据区域中的资源记录提供该区域的名称解析，因此在区域创建完成后，需要在区域中创建所需的资源记录

1. 创建主机记录

创建win2008-1主机记录

(1)以域管理员身份登录win2008-1，在DNS管理控制台中，右键正向主要区域的lyl26.com，选择新建主机，如图所示

(2)打开“新建主机”，打开“新建主机”对话框，通过此对话框可以创建A记录，如图所示

2. 创建别名记录

Win2008-2同时还是web服务器，设置别名www

(1) 选择“新建别名”

(2) 在“别名”文本框中输入规范名称，单击浏览选择起别名的目的服务器

3. 创建邮件交换器记录

(1) 选择“新建邮件交换器（MX）”

(2) 在“主机或子域”中输入MX记录的名称，如果邮件地址为yy@mail.lyl26.com,则应输入mail作为MX记录的名称记录

(3) 在“邮件服务器的完全限定的域名”文本框输入邮件服务器的名称

(4) 如果存在两个或更多的MX记录，则在解析时将首选优先级高的MX记录

4. 创建指针记录

(1) 以域管理员账户登录win2008-1，打开DNS管理控制台，选择“新建指针”

(2) 在打开的“新建资源记录”对话框的“指针（PTR）”选项卡中即可创建PTR记录

(3) 资源记录创建完后，在DNS管理控制台和区域数据库文件中都可以看到这些资源记录，如图所示

9.4 配置DNS客户端并测试主DNS服务器

9.4.1 配置DNS客户端

可以通过手动，也可以DHCP自动配置DNS客户端（要求DNS客户端是DHCP客户端）

9.4.2 测试DNS服务器

以下ip变化为192.168.26.100（对应10.10.10.1），忽略即可

1.进入nslookup测试环境

2，测试主机记录

3.测试正向解析的别名记录

4.测试MX记录

5.测试指针记录

6.查找区域信息

set type表示设置查找的类型

set type=mx，表示查找邮件服务器记录

set type=cname，表示查找别名记录

set type=A，表示查找主机记录

set type=PRT，表示查找主机记录

set type=ns，表示查找区域

9.4.3 管理DNS客户端缓存

进入cmd命令行

查看DNS客户端缓存指令：

Ipconfig /displaydns

清空DNS客户端缓存指令：

Ipconfig /flushdns

9.5.2 配置DNS转发器

1.更改该客户端DNS服务器IP地址指向

(1)登录win2008-2，将首选DNS服务器指向192.168.26.102（win2008-3）

(2)打开命令行，输入“ping win2008-2.lyl26.com”，发现命令不能解析，因为记录存在于服务器win2008-1上，不存在服务器192.168.26.102上

2.在惟缓存DNS服务器上安装DNS服务器并配置DNS转发器

(1) 以管理员身份登录将要部署惟缓存DNS服务器的计算机win2008-3

(2) 安装DNS服务

(3) 打开DNS控制台，在左侧右键DNS服务器win2008-3，在弹出的菜单选择“属性”

(4) 单击“编辑”按钮，打开“编辑转发器”对话框，在“转发服务器的IP地址”选项区域中

添加需要转发到的DNS服务器地址为192.168.26.100，单击确定即可，该计算机能解析到相应服务器FQDN

(5) 采用相同方法，根据需要，配置其他区域的转发

3.测试惟缓存DNS服务器

在win2008-2打开命令行，使用nslookup命令测试惟缓存DNS服务器
